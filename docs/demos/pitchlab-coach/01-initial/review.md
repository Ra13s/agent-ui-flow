# Initial Review

Input image: `01-initial/screenshot.png`

Page purpose: get a returning user into one daily public-speaking practice session.

## Findings

High / visual focus / the page has four attention centers / the user cannot tell whether the hero,
timer, daily-plan card, or metric cards are the real entry point / make the active daily session the
single dominant object and demote timer/metrics.

High / clutter / nearly every item is boxed, bordered, badged, or colored / the page feels busy before
the user has done anything / reduce borders, remove non-actionable badges, and keep one clear CTA.

High / information value / "confidence + filler words" is repeated in the hero, daily plan, metric
cards, and explanation blocks / repetition makes the app look confused rather than reinforcing the
lesson / say the focus once and let the drill card carry the rest.

Medium / CTA hierarchy / three hero buttons and a second orange button compete / the page asks the
user to choose between equivalent starts / keep one primary action.

Medium / AI design tells / thick side accents, symmetrical metric cards, and generic coach text feel
like generated SaaS furniture / replace them with a simpler learning-app flow.

Medium / product fit / progress metrics appear before practice starts / progress belongs on a progress
page or post-session review / keep this page about starting the practice.
