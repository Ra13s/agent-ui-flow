# Brief

Stage: brief.

## Product

PitchKit is a fictional public-speaking practice app. Users record short pitches and get feedback on
clarity, pacing, filler words, pauses, and delivery.

## Demo Goal

Demonstrate Agent UI Flow as a full hillclimb:

1. start from a mediocre real UI
2. generate multiple UI directions
3. review and select
4. generate again
5. implement
6. generate from the implementation screenshot
7. manually improve over four reviewed rounds
8. publish a visual before/after story with logs

## Initial Screen Problem

The starting page is plausible but weak:

- huge generic headline
- repeated streak/focus information
- several competing CTAs
- recap preview before the user has recorded anything
- progress/dashboard modules mixed into a start screen
- no clear first-glance object

## Constraints

- Use a fictional product.
- Do not show real private app screenshots.
- Public demo narrative should use images, not code snippets.
- Every image, review, decision, and implementation change must be logged.
