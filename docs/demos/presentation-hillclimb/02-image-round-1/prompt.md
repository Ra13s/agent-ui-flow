# Image Round 1 Prompt

Use case: ui-mockup
Asset type: desktop web app screen concept
Primary request: Redesign the provided cluttered daily page for a fictional public-speaking practice app called PitchKit.
Input image: initial screenshot is the problem reference, not a style target.
Product context: Users record short 60-second pitches and get feedback on clarity, pacing, pauses, filler words, and delivery.
Screen job: Start today's speaking practice.
What is wrong with the source: huge generic headline, repeated streak/focus info, too many CTAs, recap preview before work is done, side stats competing with the main action, weak first-glance focus.
Goal: Create a cleaner, more focused daily practice page.
Required content:
- product name: PitchKit
- daily task: "Introduce your idea to a skeptical room"
- one primary action: "Start today's practice"
- small secondary skill snapshot with Pauses, Clarity, Energy
- 4-day streak may appear once
Avoid:
- no fake analytics dashboard
- no recap preview
- no checklist
- no multiple primary CTAs
- no giant landing-page headline
- no sci-fi AI assistant panel
- no code
Visual direction: premium learning app, warm white background, generous negative space, clear focal object, coral/violet/green/yellow/teal accents used sparingly, rounded cards, not crowded.

Generate one distinct UI direction. This variant should be meaningfully different from other variants in layout strategy.
